# 过滤器 {#filters}

进一步了解Discover中可用的不同过滤器以及如何使用它们。

>[!NOTE]
>
>Discover筛选器中的“匹配用户属性”和“匹配（高级）”运算符纯粹是管理操作，可以安全地忽略它们。

**帐户Id**

_用于：基于帐户的营销_

从CRM中选择或粘贴一系列帐户ID以过滤结果。 帐户ID比帐户名称提供的唯一性更强，因为名称可以相同。

**帐户名称**

_用于：基于帐户的营销_

从CRM中选择或粘贴一系列帐户名称，以过滤结果。 字符串可以具有重复项，因此可能具有多个“[!DNL Marketo Measure]”帐户。 在这种情况下，如果需要单个帐户，请改用帐户ID过滤器。

**归因模型**

_用于：概述，营销支出，广告ROI，基于帐户的营销， Web流量， CMO，付费媒体，内容营销， Passport_

选择要应用于展示板的单个归因模型：首次接触、潜在客户创建接触、U型、W型、完整路径或自定义模型。 并非在所有层中都提供完整路径和自定义模型。

**Campaign**

_用于：概述、增长、广告ROI、Web流量、CMO、付费媒体、内容营销、护照_

按单个或多个营销活动名称过滤展示板。 运算符为过滤器提供了额外的灵活性，例如使用“包含”或“开头为”运算符。 如果已应用渠道或子渠道过滤器，则显示的营销活动列表将是已应用过滤器的子集。

**第1-10类**

_用于：概述、增长、广告ROI、CMO、付费媒体、内容营销、Velocity、快照、同类群组漏斗、Passport_

使用您在 [!DNL Marketo Measure] 设置。 您创建的类别列表将显示在过滤器菜单中，因此，如果未设置类别，则菜单中将不会设置类别过滤器。 区段类别并非在所有层中都可用，可用的类别数量也因层而异。

**渠道**

_用于：概述、增长、营销支出、广告ROI、Web流量、CMO、付费媒体、内容营销、Velocity、Passport_

按单个或多个渠道过滤展示板。 运算符为过滤器提供了额外的灵活性，例如使用“包含”或“开头为”运算符。 输入渠道后，子渠道和促销活动过滤器中显示的值将来自应用的子渠道过滤器。

**同类群组阶段**

_用于：同类群组漏斗_

选择要查看同类群组的阶段。 您选择的阶段将显示在漏斗顶部，所有转化都从顶部向下流动。

**日期**

_用于：概述、增长、营销支出、广告ROI、基于帐户的营销、Web流量、CMO、付费媒体、内容营销、Velocity、快照、同类群组漏斗、Passport_

选择一个日期范围以使用灵活的日期运算符（例如“在范围内”、“在年份中”或“在之前”）过滤展示板中的数据。 快照除外，在快照中，您将选择一个日期来查看数据的快照。

**日期类型**

_用于：概述、增长、营销支出、广告ROI、基于帐户的营销、Web流量、CMO、付费媒体、内容营销、护照_

选择要使用的日期类型，并将其绑定到日期过滤器。 默认日期类型因展示板而异。 Touchpoint Date是指营销活动发生的日期， Created Date是在CRM中创建Lead或Contact或Opportunity的日期， Close Date是Opportunity的关闭日期。

**维度**

_用于：付费媒体_

Dimension与按组功能类似，只是其在付费媒体板上的使用方式略有不同。 Dimension不会堆叠图表，而是会更改“概述”图表中的线条以及表中的前导对象。

![](assets/1.png)

默认情况下，“Dimension”设置为“子渠道”，并可以将其更改为：

* 无：显示聚合中的所有内容，不进行划分
* 渠道：按营销渠道列出数据
* 子渠道：按营销子渠道列出数据
* 营销活动：按营销活动列出数据
* 帐户：按帐户列出数据。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 广告组：按广告组列出数据。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 广告：按广告列出数据。 适用于Doubleclick广告，因此如果不使用Doubleclick，则不会显示任何结果
* 广告商：按广告商列出数据。 适用于Doubleclick广告商，因此如果不使用Doubleclick，则不会显示任何结果
* 创意：按创意列出数据。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 关键词：按关键词列出数据。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 版面：按放置列出数据。 适用于Doubleclick版面，因此如果不使用Doubleclick，则不会显示任何结果
* 网站：按站点列出数据。 适用于Doubleclick站点，因此如果不使用Doubleclick，则不会显示任何结果

**分组依据**

_用于：概述、增长、营销支出、基于帐户的营销、Web流量、CMO_

调整图表以更改正在堆叠并分组在一起的维度。 默认情况下，“分组依据”设置为“渠道”，并可更改为：

* 无：显示聚合中的所有内容，不进行划分
* 渠道：按营销渠道对数据进行分组
* 子渠道：按营销子渠道对数据进行分组
* 营销活动：按营销活动对数据进行分组
* 帐户：按帐户对数据进行分组。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 广告组：按广告组对数据进行分组。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 广告：按广告对数据进行分组。 适用于Doubleclick广告，因此如果不使用Doubleclick，则不会显示任何结果
* 广告商：按广告商对数据进行分组。 适用于Doubleclick广告商，因此如果不使用Doubleclick，则不会显示任何结果
* 创意：按创意对数据进行分组。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 关键词：按关键词对数据进行分组。 适用于 [!DNL AdWords], [!DNL Bing]和 [!DNL Facebook].
* 版面：按放置对数据进行分组。 适用于Doubleclick版面，因此如果不使用Doubleclick，则不会显示任何结果
* 网站：按站点对数据进行分组。 适用于Doubleclick站点，因此如果不使用Doubleclick，则不会显示任何结果

**登陆页面**

_用于：内容营销_

深入了解单个登陆页面，或可能包含特定词语（如“blog”）的登陆页面的性能。

**量度**

_用于：概述， Web流量， CMO，付费媒体，内容营销_

在不同的展示板中使用两个不同的量度选取器。 量度选取器会更改图表上的度量，以便您可以切换查看收入或支出或展示次数等。

在“概述”和CMO展示板上，提供了与ROI量度相关的缩短值列表：

* 收入
* 支出
* 交易
* 管道收入
* 机会
* 联系人
* 潜在客户

在Web流量、付费媒体和内容营销展示板上，有更长的与ROI和漏斗量度相关的值列表：

* 收入
* 支出
* 交易
* 管道收入
* 机会
* 联系人
* 潜在客户
* 点击次数
* 展示次数
* 访问次数
* 独特访问量
* 页面查看次数
* Forms

**阶段**

_用于：速度_

默认情况下，Velocity展示板会显示所有阶段的时间，但要深入到特定阶段，请使用“阶段”筛选器选择该阶段。

**子渠道**

_用于：概述、增长、营销支出、广告ROI、Web流量、CMO、付费媒体、内容营销、护照_

按单个或多个子渠道过滤展示板。 运算符为过滤器提供了额外的灵活性，例如使用“包含”或“开头为”运算符。 如果已应用渠道过滤器，则显示的子渠道列表将是已应用过滤器的子集。 输入子渠道后，营销活动过滤器中显示的值将来自应用的子渠道过滤器。

**URL**

_用于：Web流量_

深入研究单个URL的流量，或可能包含某个词语（如“product”）的URL的流量。

**元**

_用于：速度_

默认情况下，Velocity展示板仅报告已结束的赢取机会，但调整此过滤器以查看已结束的赢取机会或已结束的丢失机会的速度。
